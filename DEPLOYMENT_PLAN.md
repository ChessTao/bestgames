# План публикации страницы на сайте

Проект можно публиковать как статическую страницу: достаточно загрузить HTML,
CSS, JavaScript, локальные библиотеки и PGN-данные на любой обычный хостинг.

## 1. Что публикуем

Основная точка входа:

- `index.html`

Файлы, которые подключает страница:

- `Styles.css`
- `StockfishEngine.js`
- `app-pgn.js`
- `app-position.js`
- `app-render.js`
- `App.js`
- `vendor/chess/chess-0.10.3.min.js`
- `vendor/stockfish/stockfish-17.1-lite-single-03e3232.js`
- `vendor/stockfish/stockfish-17.1-lite-single-03e3232.wasm`
- `vendor/pieces/cburnett/*.svg`
- `storage/games/best games.pgn`

Готовый комплект уже лежит в папке `deploy-package/`. Для ручной загрузки можно
использовать именно её содержимое или архив `deploy-package.zip`.

## 2. Рекомендуемое место на сайте

Если страница должна быть отдельным разделом сайта, загрузить файлы в папку:

```text
/public_html/best-games/
```

После этого страница будет открываться по адресу вида:

```text
https://aveskulov.com/best-games/
```

Если страница должна стать главной страницей сайта, содержимое `deploy-package/`
нужно загрузить прямо в корень публичной директории, например:

```text
/public_html/
```

## 3. Подготовка перед загрузкой

Перед публикацией проверить локально:

- `index.html` открывается без ошибок;
- шахматная доска отображается;
- фигуры загружаются из `vendor/pieces/cburnett/`;
- партии подгружаются из `storage/games/best games.pgn`;
- анализ Stockfish запускается и не даёт ошибок загрузки `.js` или `.wasm`;
- в консоли браузера нет ошибок JavaScript;
- все пути в `index.html` относительные, без привязки к локальному диску.

## 4. Загрузка на хостинг

Вариант через файловый менеджер хостинга:

1. Открыть файловый менеджер панели хостинга.
2. Перейти в нужную публичную папку сайта.
3. Создать папку `best-games`, если страница публикуется как раздел.
4. Загрузить содержимое `deploy-package/` или распаковать туда `deploy-package.zip`.
5. Убедиться, что структура папок сохранилась.

Итоговая структура в папке публикации должна быть такой:

```text
best-games/
  index.html
  Styles.css
  StockfishEngine.js
  app-pgn.js
  app-position.js
  app-render.js
  App.js
  storage/
    games/
      best games.pgn
  vendor/
    chess/
      chess-0.10.3.min.js
    pieces/
      cburnett/
        *.svg
    stockfish/
      stockfish-17.1-lite-single-03e3232.js
      stockfish-17.1-lite-single-03e3232.wasm
```

## 5. Проверка после публикации

Открыть опубликованный адрес и проверить:

- страница загружается без `404`;
- CSS применился, интерфейс выглядит как локально;
- доска, фигуры, список партий и ходы работают;
- переходы по ходам не ломаются;
- варианты партий отображаются корректно;
- Stockfish загружается и показывает анализ;
- в DevTools во вкладках Console и Network нет ошибок загрузки CSS, JS, SVG, PGN и WASM.

## 6. Что не обязательно загружать

На публичный сайт не нужны служебные файлы разработки:

- `DEPLOYMENT_PLAN.md`
- `SMOKE_CHECK.md`
- `smoke-check.js`
- `server.js`
- `.git/`
- старые HTML-варианты, если они не используются

Их можно оставить локально в проекте.

## 7. Быстрый итог

Для публикации достаточно загрузить на хостинг содержимое `deploy-package/`.
Главное сохранить относительную структуру папок, потому что страница ищет
библиотеки, фигуры, Stockfish и PGN-файл по текущим путям.
